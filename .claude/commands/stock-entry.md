# stock-entry

Generate a specific entry/exit point analysis for a ticker: entry zone, stop loss, price targets, position sizing, and key option market levels. Writes a dated file to the ticker's wiki directory and updates changelog and overview.

## Arguments

Required: `<TICKER>` — the stock symbol (e.g. `POET`, `NVDA`).

## Steps

### 1. Normalize and check existing research

Normalize `$ARGUMENTS` to uppercase before use in all file paths and headers.

Read `wiki/tickers/$ARGUMENTS/overview.md` if it exists. Extract current thesis status (Active/Watch/Avoid) and any prior entry analyses linked in the "历史入场分析" section. Tell the user what you found.

If `wiki/tickers/$ARGUMENTS/` does not exist at all, warn: "No existing research found for $ARGUMENTS. Consider running `/stock-analyze $ARGUMENTS` first for a complete thesis. Proceeding with entry analysis only."

### 2. Technical position (SEPA)

Use the `sepa-strategy` skill to determine:
- Current Stage (1 / 2 / 3 / 4)
- Trend template score (X/7 criteria passing)
- Price vs 50MA / 150MA / 200MA (exact values)
- ATR (14-day Average True Range)
- Current pattern if any (VCP, flat base, cup-with-handle, bull flag, none)
- Pivot point / ideal entry price
- Whether current setup is Actionable / Watch / Not Ready

### 3. Current price and volume

Use the `yfinance-data` skill to get:
- Current price (or last close)
- Today's volume vs 50-day average volume (volume ratio)
- 52-week high and low
- Distance from 52-week high (%)

### 4. Liquidity check

Use the `stock-liquidity` skill to estimate:
- ADTV (average daily dollar volume)
- Estimated slippage for a $25K position

### 5. Option market levels

Query the unusual_whales Telegram bot (Chat ID: 5241900942) for recent messages:

```bash
tdl chat export -c 5241900942 -T last -i 5 --all --with-content -o ~/tdl-exports/uw_entry.json
py scripts/read_uw_bot.py --path ~/tdl-exports/uw_entry.json --ticker $ARGUMENTS --limit 5
py scripts/read_uw_bot.py --path ~/tdl-exports/uw_entry.json --section flow --limit 5
```

Extract any relevant data for `$ARGUMENTS` from helper output. Also note: the user can manually send `/max_pain $ARGUMENTS` and `/oi_strike $ARGUMENTS` to the bot in the Telegram app for precise OI data.

Report:
- Max pain strike for the nearest expiry (if available from bot data)
- Nearest OI wall above current price (resistance, if available)
- Nearest OI wall below current price (support, if available)

If the bot data is unavailable or contains no relevant data for `$ARGUMENTS`, note "OI data unavailable" and continue.

### 6. Options payoff visualization (optional)

If the user has mentioned an options strategy (calls, puts, spreads) before running this command, use the `options-payoff` skill to visualize the P&L curve.

Skip this step if the user only wants stock entry analysis.

### 7. Synthesize and present entry plan

Present the following structured output and ask: "确认写入文件？" Wait for an explicit affirmative (e.g., "yes", "ok", "确认", "写入") before proceeding to Step 8. If the user requests changes, revise the plan and re-present Step 7. Do not write any files until confirmed.

```
## Entry Analysis — $ARGUMENTS — [TODAY'S DATE]

**Current Price:** $XX.XX
**Technical Stage:** Stage X ([description])
**Setup Quality:** [Actionable / Watch / Not Ready]
**Pattern:** [VCP / flat base / none / etc.]

### Entry Plan
入场区间：$XX.XX — $XX.XX
（理由：[1 sentence — e.g. "pivot breakout above $XX.XX with volume confirmation"]）

止损位：$XX.XX（风险：X% from entry midpoint）
（理由：[1 sentence — e.g. "below 50MA at $XX.XX"]）

目标价 1：$XX.XX（盈亏比 X:1，理由：[prior resistance / measured move / etc.]）
目标价 2：$XX.XX（盈亏比 X:1，理由：[52-week high / analyst target / etc.]）

### Position Sizing（基于 1% 本金风险）
- $50K 组合 → 最大仓位 $X,XXX（约 XXX 股）
- $100K 组合 → 最大仓位 $X,XXX（约 XXX 股）
- $200K 组合 → 最大仓位 $X,XXX（约 XXX 股）

### Key Levels
ATR (14d)：$X.XX
52周高：$XX.XX（距今 X%）
52周低：$XX.XX
期权 Max Pain：$XX（[到期日] 或 N/A）
OI 阻力位：$XX.XX（或 N/A）
OI 支撑位：$XX.XX（或 N/A）

### Execution Note
ADTV：$XXM
预计滑点（$25K 仓位）：约 $XX
```

### 8. Write entry file

Use YYYY-MM-DD format for all date placeholders in this command (filenames, headers, changelog entries, and the completion report).

After user confirmation, write `wiki/tickers/$ARGUMENTS/entry-[TODAY'S DATE].md`:

Create `wiki/tickers/$ARGUMENTS/` if it does not already exist.

### 9. Update changelog

If `wiki/tickers/$ARGUMENTS/changelog.md` exists, prepend a new entry below the file header and before existing entries (newest-on-top):

```markdown
## [TODAY'S DATE] — Entry Analysis

**入场区间：** $XX.XX — $XX.XX | **止损：** $XX.XX（X%）| **目标 1：** $XX.XX | **目标 2：** $XX.XX
**Setup：** Stage X，[pattern or none]
**File：** [entry-[TODAY'S DATE].md](entry-[TODAY'S DATE].md)

---
```

If `changelog.md` does not exist, create it with this header followed by the entry:

```markdown
# $ARGUMENTS — Changelog

Per-ticker change log documenting thesis evolution and action triggers.
**Language**: English | [中文](changelog.zh.md)

---
```

### 10. Update overview

Read `wiki/tickers/$ARGUMENTS/overview.md`. Find the section `## 历史入场分析`. If it exists, prepend a new link (newest first). If it does not exist, create the section after the last existing section:

```markdown
## 历史入场分析

- [[TODAY'S DATE] — 入场 $XX.XX–$XX.XX，止损 $XX.XX](entry-[TODAY'S DATE].md)
```

### 11. Confirm completion

Report:
```
入场分析完成：
- wiki/tickers/$ARGUMENTS/entry-[TODAY'S DATE].md 已创建
- changelog.md 已更新
- overview.md 已添加链接
```
