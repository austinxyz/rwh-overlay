# rwh-overlay

Personal overlay on top of [kgajjala/rwh](https://github.com/kgajjala/rwh) — Austin's
investment-analysis extensions built on kgajjala's framework.

## What lives here
- `wiki/tickers/` — my own ticker analyses (outside kgajjala's coverage)
- `wiki/opinions/` — third-party observation sources (e.g., Yun Chen WeChat group)
- `raw/analyses/` — raw source material I author (e.g., Chen timeline)
- `scripts/` — build pipeline that merges upstream + overlay into a published tree
- `.claude/commands/` — Claude slash commands for research workflows
- `tools/` — finance skills documentation and tool references
- `CLAUDE.overlay.md` — overlay-specific rules layered on top of upstream `CLAUDE.md`

## What does NOT live here
- kgajjala's tickers (BKNG, LLY, WING, ...) — stay in `../rwh/` (read-only mirror)
- Derived files (index.md, watchlist.md, log.md) — regenerated by build script
- The build output `../stock-kb/` — plain directory, no git, fully regenerated on each build
- `data/positions.md` — local position tracker (gitignored, maintained by `/morning-check`)

## How to build

```
py scripts/build_stock_kb.py   # or /kb-sync inside Claude
```

Requires: `py`/`python` and `python-frontmatter` installed. No `rsync` or bash shell needed.

## Claude Slash Commands

| Command | Purpose |
|---------|---------|
| `/kb-sync` | Build and sync `../stock-kb/` from upstream + overlay |
| `/stock-analyze <TICKER>` | Deep-dive analysis producing full 15-section thesis files |
| `/stock-refresh <TICKER>` | Quick refresh when new info arrives (earnings, filing, news) — updates only changed sections |
| `/stock-entry <TICKER>` | Entry/exit point analysis (zones, stop, targets, options levels) |
| `/morning-check <TICKER>` | Open-time decision: compare live price to entry plan, output Buy/Wait/Skip/Trim/Exit recommendation |
| `/morning-check ALL` | Batch scan all active positions (from `data/positions.md`) |
| `/chen-integrate` | Parse `raw/analyses/chen.md`, write daily log, flag tickers to validate |
| `/chen-validate <TICKER>` | Cross-validate a Chen-mentioned ticker via sentiment/options/technicals |
| `/market-daily` | Generate daily market report |
| `/market-weekly` | Generate weekly market recap |
| `/market-monthly` | Generate monthly market recap |
| `/market-quarterly` | Generate quarterly market recap |

## Day-to-day
- Add new ticker: `/stock-analyze <TICKER>` → commit + push.
- Refresh on earnings/news: `/stock-refresh <TICKER> [trigger]` → commit + push.
- Add entry plan: `/stock-entry <TICKER>` → commit + push.
- Update Chen observations: append to `raw/analyses/chen.md` → `/chen-integrate` → optionally `/chen-validate <TICKER>`.
- Market recap: `/market-daily` (or weekly/monthly/quarterly) after market close.
- Morning open decision: `/morning-check <TICKER>` (single deep) or `/morning-check ALL` (batch positions scan).
- Position tracking: `data/positions.md` (gitignored, maintained by `/morning-check`).
- Rebuild KB: `/kb-sync` (pulls upstream automatically).

See `docs/requirements.md` for full feature specs.
